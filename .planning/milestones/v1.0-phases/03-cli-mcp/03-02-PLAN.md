Register all 5 tools using `server.tool(name, description, zodRawShape, handler)` pattern. Use raw Zod shape (NOT z.object()). Keep descriptions concise -- MCP-09 requires total schema under 4000 chars. Target single-sentence descriptions, parameter descriptions under 60 chars.

**Tool schemas:**

1. `graft_map`: "Ranked tree map of the codebase by structural importance."
   - `query`: `z.string().optional().describe('File or symbol for personalization')`
   - `budget`: `z.number().int().min(1).max(32000).optional().describe('Max tokens (default 2048)')`

2. `graft_context`: "Dependencies and definitions for a specific file."
   - `path`: `z.string().describe('File path relative to project root')`

3. `graft_search`: "Find definitions by name or kind."
   - `query`: `z.string().describe('Name pattern to search for')`
   - `kind`: `z.string().optional().describe('Filter by kind: function, class, type, etc.')`

4. `graft_impact`: "Files affected by changing a given file."
   - `path`: `z.string().describe('File path relative to project root')`

5. `graft_summary`: "Project overview with key files and tech stack."
   - (no params)

**Tool handler implementations:**

For `graft_map`:
- If query provided, resolve it against rootDir to get absolute path. Create `personalization = new Map([[absPath, 10.0]])`. Pass to `buildIndex(rootDir, personalization)`.
- If no query, call `buildIndex(rootDir)`.
- Call `renderTree(graph, scores, rootDir, { tokenBudget: budget ?? 2048, charsPerToken: 3 })`.
- Return `{ content: [{ type: 'text' as const, text: output }] }`.

For `graft_context`:
- Resolve `path.resolve(rootDir, params.path)` to absPath.
- Call `buildIndex(rootDir)`.
- Get definitions: `graph.definitions.get(absPath) ?? []`, filter out import/export kinds.
- Get forward: `forwardDeps(graph, absPath).files`.
- Get reverse: `reverseDeps(graph, absPath).files`.
- Format as text sections: "## Definitions\n...", "## Dependencies\n...", "## Dependents\n...".
- Use `path.relative(rootDir, ...)` for all displayed paths.
- Return text content.

For `graft_search`:
- Call `buildIndex(rootDir)`.
- Iterate all `graph.definitions` entries, for each file's definitions filter by `def.name.toLowerCase().includes(query.toLowerCase())`.
- If `kind` param provided, additionally filter by `def.kind === kind`.
- Format each match as `"{kind} {name} in {relativePath}:L{startLine}"`.
- Return text content with match count header.

For `graft_impact`:
- Resolve abs path.
- Call `buildIndex(rootDir)`.
- Call `transitiveClosure(graph, absPath)`.
- Format each affected file with its score, sorted by score descending.
- Return text content.

For `graft_summary`:
- Call `buildIndex(rootDir)`.
- Compute: `graph.files.size` (file count), sum of all definitions arrays lengths (def count), sum of forwardEdges set sizes (edge count).
- Sort files by score, take top 10.
- Try `fs.readFileSync(path.join(rootDir, 'package.json'))` in a try/catch. If exists, parse JSON and extract `Object.keys(deps)` for tech stack. If not found, show "No package.json detected".
- Format as structured text sections: "## Project Stats\n...", "## Key Files\n...", "## Tech Stack\n...".
- Return text content.

CRITICAL: Never use console.log anywhere in this file. Use `import { info, warn } from '../logger.js'` for any logging (all goes to stderr).

Write tests in `tests/mcp/tools.test.ts` that mock `buildIndex`, `renderTree`, `forwardDeps`, `reverseDeps`, `transitiveClosure` to test each tool handler's logic in isolation. Do NOT test McpServer/StdioServerTransport connection (that is covered by Task 3). Test the handler functions by extracting them or by testing the formatting/logic directly.

Approach for testability: Extract each tool's handler logic into a named async function (e.g., `handleGraftMap`, `handleGraftSearch`, etc.) that takes params + rootDir and returns the MCP content response. Register these with `server.tool()`. Test the handler functions directly.

server.resource(
  'map',
  'graft://map',
  { description: 'Full ranked codebase tree map' },
  async (_uri) => {
    const { graph, scores } = await buildIndex(rootDir);
    const text = renderTree(graph, scores, rootDir, { tokenBudget: 8192, charsPerToken: 3 });
    return { contents: [{ uri: 'graft://map', text, mimeType: 'text/plain' }] };
  },
);
