  <files>src/graph/types.ts, src/graph/index.ts, tests/graph/build.test.ts</files>
  <behavior>
    - Test: buildGraph with empty ParseResult[] returns FileGraph with 0 files, empty edges
    - Test: buildGraph with 3 files (A imports B, B imports C) produces correct forward edges (A->{B}, B->{C}, C->{}) and reverse edges (A->{}, B->{A}, C->{B})
    - Test: Import references to unknown files (npm packages like 'lodash') are silently dropped, no edges created
    - Test: Relative import paths (../utils, ./models) resolve to absolute paths that match files set entries
    - Test: Python relative imports (.models, ..core) resolve correctly by counting dots
    - Test: Barrel file re-exports (export kind nodes with references) create edges
    - Test: Duplicate imports from same file produce single edge (Set deduplication)
    - Test: File with only definitions and no imports has empty forward edges but appears in files set
    - Test: Index file resolution (./foo resolves to ./foo/index.ts when ./foo.ts not in files set)
  </behavior>
  <action>
    1. Create `src/graph/types.ts` with ALL Phase 2 type contracts (these are the interface contracts for the entire phase):
       - `FileGraph` interface: `files: ReadonlySet<string>`, `forwardEdges: ReadonlyMap<string, ReadonlySet<string>>`, `reverseEdges: ReadonlyMap<string, ReadonlySet<string>>`, `definitions: ReadonlyMap<string, readonly CodeNode[]>`
       - `PageRankOptions` interface: `alpha: number`, `maxIterations: number`, `tolerance: number`, `personalization?: ReadonlyMap<string, number>`
       - `PageRankResult` interface: `scores: ReadonlyMap<string, number>`, `iterations: number`, `converged: boolean`
       - `TreeRendererOptions` interface: `tokenBudget: number`, `charsPerToken: number`
       - `JsonRendererOptions` interface: (empty for now, reserved for Phase 2 plan 03)
       - `TraversalResult` interface: `files: ReadonlySet<string>` (for forward/reverse/transitive queries)

    2. Create `src/graph/index.ts` with `buildGraph(results: readonly ParseResult[]): FileGraph`:
       - Index all files from results into `files` Set
       - Initialize empty forward/reverse edge Sets for every file
       - Store definitions per file
       - For each import/export node in each result, iterate `node.references`:
         - Call `resolveImportPath(importerFilePath, ref, knownFiles)` to resolve relative paths
         - If resolved and in files set, add forward edge from importer to resolved, reverse edge from resolved to importer
       - `resolveImportPath` implementation: skip non-relative imports (no leading `.`), use `path.resolve(importerDir, ref)`, try bare path then extensions [.ts, .tsx, .js, .mjs, .cjs, .py], then index variants [/index.ts, /index.js, /index.py]
       - For Python relative imports: count leading dots, navigate up that many directories from importer, then resolve the remainder

    3. Create `tests/graph/build.test.ts` with tests per behavior spec above. Use synthetic ParseResult objects — do NOT depend on real file parsing. Test file paths should use `/project/src/` prefix to be platform-agnostic.
  </action>

  </verify>
