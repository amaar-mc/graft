<task type="auto">
  <name>Task 2: Create ESLint config, logger, error classes, and CodeNode types</name>
  <files>eslint.config.ts, src/logger.ts, src/errors.ts, src/parser/types.ts</files>
  <action>
Create `eslint.config.ts` — ESLint 9 flat config:
- For `src/**/*.ts`: enable `@typescript-eslint/no-explicit-any: "error"`, `@typescript-eslint/explicit-function-return-type: "error"`, `no-console: "error"`
- For `src/cli/**/*.ts`: override `no-console: "off"` (CLI code may use console for user-facing output)
- Use `@typescript-eslint/parser` as the language parser
- Reference research Pattern (ESLint 9 flat config example) for exact syntax. Note: ESLint 9 flat config API may differ from research example — check the actual installed version's API. Use `defineConfig` from `eslint/config` if available, otherwise use plain array export.

Create `src/logger.ts`:
- Export a `log` function with signature: `log(level: LogLevel, message: string, meta?: Record<string, unknown>): void`
- `LogLevel` type: `'debug' | 'info' | 'warn' | 'error'`
- All output goes to `process.stderr.write()` — never `console.log()`, never `process.stdout`
- Format: `[graft:${level}] ${message}` with optional JSON meta appended
- Export convenience functions: `debug(msg, meta?)`, `info(msg, meta?)`, `warn(msg, meta?)`, `error(msg, meta?)`

Create `src/errors.ts`:
- Export `GraftError extends Error` with a `readonly hint: string` field and `readonly code: string` field
- Export specific subclasses: `ParseError` (code: "PARSE_ERROR"), `DiscoveryError` (code: "DISCOVERY_ERROR"), `GrammarLoadError` (code: "GRAMMAR_LOAD_ERROR")
- Each constructor takes `message: string` and `hint: string` — hint tells the user what to do
- Example: `new GrammarLoadError("Failed to load TypeScript WASM grammar", "Run 'pnpm install' to ensure grammar files are present, or check that .wasm files exist in dist/")`

Create `src/parser/types.ts`:
- Export `NodeKind` type: `'function' | 'method' | 'class' | 'interface' | 'type' | 'enum' | 'module' | 'constant' | 'import' | 'export' | 'decorator' | 'variable'`
- Export `CodeNode` interface with all readonly fields: `id` (string, format `${filePath}:${name}:${startLine}`), `name` (string), `kind` (NodeKind), `filePath` (string, absolute path), `startLine` (number), `endLine` (number), `references` (readonly string[] — names this node references)
- Export `ParseResult` interface: `{ readonly filePath: string; readonly nodes: readonly CodeNode[]; readonly parseTimeMs: number }`
- Export `LanguageId` type: `'typescript' | 'tsx' | 'javascript' | 'python'`
- Export `fileExtensionToLanguage` function that maps: `.ts` -> `'typescript'`, `.tsx` -> `'tsx'`, `.js`/`.mjs`/`.cjs` -> `'javascript'`, `.py` -> `'python'`, else -> `null`
