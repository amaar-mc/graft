Create `src/indexer/discovery.ts`:

Use `fast-glob` for async recursive enumeration and `ignore` package for gitignore-compatible filtering. Follow research Pattern 5 closely.

Function signature:
```typescript
export async function discoverFiles(
  rootDir: string,
  extraIgnorePatterns?: readonly string[]
): Promise<string[]>
```

Default ignore list (hardcoded, always applied):
```
node_modules, dist, build, .next, out, coverage,
__pycache__, .venv, venv, .git, vendor,
*.min.js, *.d.ts, *.map, *.bundle.js
```

Logic:
1. Validate `rootDir` exists — if not, throw `DiscoveryError` with hint "Check that the directory path is correct and exists"
2. Create `ignore` instance, add default patterns
3. Read `.gitignore` from rootDir if present, add to ignore instance
4. Read `.graftignore` from rootDir if present, add to ignore instance
5. Add `extraIgnorePatterns` if provided
6. Run `fast-glob` with pattern `**/*.{ts,tsx,js,mjs,cjs,py}`, `cwd: rootDir`, `absolute: true`, `dot: false`, `followSymbolicLinks: false`
7. Filter results through `ignore` instance using relative paths
8. Sort results alphabetically for deterministic output
9. Log discovery stats via `logger.info` (file count, time elapsed)

Use `logger.debug` for ignored-file logging. Never use `console.log`.
