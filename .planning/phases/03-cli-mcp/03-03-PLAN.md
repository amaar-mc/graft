**serve command (CLI-01 + CLI-07):**
```typescript
program
  .command('serve', { isDefault: true })
  .description('Index and start MCP server over stdio')
  .action(async () => {
    const { startMcpServer } = await import('../mcp/server.js');
    await startMcpServer(process.cwd());
  });
```
Use dynamic import for startMcpServer to avoid pulling MCP SDK into non-serve commands. `isDefault: true` makes this run when no subcommand given (CLI-07).

**map command (CLI-02):**
- Show ora spinner on stderr: `ora({ text: 'Indexing...', stream: process.stderr }).start()`
- If `--focus` provided, resolve to absolute path, create `personalization = new Map([[absPath, 10.0]])`
- Call `buildIndex(rootDir, personalization)`
- Call `renderTree(graph, scores, rootDir, { tokenBudget: parseInt(options.budget, 10), charsPerToken: 3 })`
- Spinner succeed with chalk.green summary
- Write tree output to `process.stdout.write(output + '\n')`

**stats command (CLI-03):**
- Show spinner on stderr
- Call `buildIndex(rootDir)`
- Compute: fileCount = graph.files.size, defCount = sum of definitions array lengths (exclude import/export kinds), edgeCount = sum of forwardEdges set sizes
- Read cache via `readCache(rootDir)` for `createdAt` timestamp. If cache exists, show age as relative time (e.g., "2 minutes ago"). If no cache, show "No cache".
- Format with chalk: section headers in bold, numbers in cyan
- Write to stdout

**impact command (CLI-04):**
- Takes `<path>` required argument
- Resolve to absolute: `path.resolve(rootDir, filePath)`
- Show spinner on stderr
- Call `buildIndex(rootDir)`, then `transitiveClosure(graph, absPath)`
- Sort affected files by score descending
- Format with chalk: file path, score, total count header
- Write to stdout

**search command (CLI-05):**
- Takes `<query>` required argument
- Optional `--kind <kind>` filter
- Show spinner on stderr
- Call `buildIndex(rootDir)`
- Iterate `graph.definitions`, filter by name.toLowerCase().includes(query.toLowerCase())
- If `--kind`, also filter by `def.kind === kind`
- Format: `{kind} {name} in {relativePath}:L{startLine}` with chalk colors (kind in yellow, name in white, path in dim)
- Write to stdout

**Error handling at bottom:**
```typescript
program.parseAsync(process.argv).catch((err: unknown) => {
  if (err instanceof GraftError) {
    process.stderr.write(chalk.red(`Error: ${err.message}\n`));
    process.stderr.write(chalk.yellow(`Hint: ${err.hint}\n`));
  } else {
    process.stderr.write(chalk.red(`Error: ${String(err)}\n`));
  }
  process.exit(1);
});
```

CRITICAL: All ora spinners MUST use `{ stream: process.stderr }`. All logging via stderr. Only final command output (tree, stats table, impact list, search results) goes to stdout. This prevents stdout contamination when running as MCP server.

CRITICAL: Never use console.log. Use process.stdout.write for intentional output, process.stderr.write or logger for everything else.
